## Что такое `topn` (важно — это меняет постановку)

`UNIVERSE_TOP_N_LIQUID` (`config.py`) — это **ликвидностный фильтр вселенной**, а
не параметр формулы. На месяце t: из бумаг с 13 закрытиями оставляем N самых
ликвидных по медиане месячного оборота за 12 мес (`universe.universe_at`), затем
скорим **оставшиеся** и режем на квартили (`backtest.quartile_split`, Q1 = верхние ~N/4).

**Скор бумаги от `topn` не зависит.** `(a·r(12-1)+b·r(6-1))/σ(12)` считается
по-бумажно, по её собственной истории (`signals.CurveFitSignal.compute`, `apply(axis=0)`),
без кросс-секционной нормировки. Одна и та же бумага получает идентичный скор хоть
во вселенной из 20, хоть из 120 имён. `topn` влияет ТОЛЬКО на (1) состав пула-кандидатов
и (2) где ляжет квартильная граница.

## Ответ на вопрос «имеет ли смысл второй подход»

Да, и он **чище первого**. Причина — ровно в том, что скор инвариантен к вселенной:

- **Подход 1 (фильтр в начале):** `universe_top_n ∈ {20,30,…,120}`. При уменьшении topn
  одновременно растёт ликвидностный пол И уменьшается размер Q1 (Q1 ≈ topn/4: 5 имён при
  topn=20, 30 при 120). Веер мешает две переменные — нельзя сказать, что дало эффект:
  концентрация или отсечение менее ликвидных имён. **Конфаунд.**
- **Подход 2 (фильтр в конце):** `topn=100` фиксирован, держим только top-K по скору,
  K ∈ {5,10,15,20,25}. Ликвидностный пол фиксирован, варьируется ТОЛЬКО число имён.
  Чистый ответ на «насколько концентрировать Q1».

**Они не эквивалентны при равном числе имён.** Подход-2 top-5 = 5 самых моментум-сильных
среди 100 самых ликвидных (может включать имена с ликвидностью на местах 21–100).
Подход-1 top-5 (topn=20 → Q1=5) — только среди 20 самых ликвидных. Разные портфели,
разный профиль ликвидности.

Подход 2 = «верхние K по скору из вселенной topn=100», без привязки к квартильной
границе. При K≤25 это вложенный срез Q1; K=25 = текущая прод-стратегия (полный Q1);
K=30 захватывает первые 5 имён из Q2. Квартили тут роли не играют — это просто
«держать K самых моментум-сильных ликвидных имён».

## Деливерабл

### 1. Веер 1 — ширина вселенной (подход 1)
11 NAV-кривых Q1 для `universe_top_n ∈ {20,30,…,120}` + MCFTRR. `topn=100` подсвечен
как baseline. Сигнал — `curve_fit` (прод). Механика — существующий `backtest(universe_top_n=…)`,
ничего не дописываем в движок.

### 2. Веер 2 — концентрация (подход 2)
11 NAV-кривых для top-K по скору из фиксированной вселенной topn=100,
`K ∈ {5,8,10,13,15,18,20,23,25,28,30}` (сетка по 5 + промежуточные) + MCFTRR.
Веса — equal-weight 1/K. K=25 совпадает с baseline (полный Q1) — подсветить.

Движок сейчас умеет только квартили. Нужен селектор «top-K по скору» из вселенной.
Не раздувать `backtest()` — в research-скрипте переиспользовать публичные хелперы
(`universe_at`, `signal.compute`, `gross_return`, `turnover` — module-level в `backtest.py`),
веса строить inline (equal-weight `1/K`). NAV-цикл повторяет логику `backtest`: применить
прошлые веса к доходности месяца → ребаланс по top-K → списать `commission_per_side·turnover`.

### 3. Оборот (минимизировать — главная цель пользователя)
Для каждого варианта обоих вееров посчитать:
- **замен имён/месяц** — `|new_set \ old_set|` (среднее по месяцам), в штуках и в % от K;
- **weight-turnover/месяц** — `Σ|Δw|` (это драйвер стоимости, уже есть `turnover()`);
- **годовой cost-drag** — суммарные комиссии в % NAV/год.

Гипотеза для проверки: при малом K один свап = большой вес-оборот (2/K), но число свапов
может быть меньше. Что реально минимизирует оборот — мерить, не угадывать.

### 4. Статистическая значимость превосходства над `topn=100`
Baseline = прод-стратегия = Q1, topn=100, полный (~25 имён), curve_fit. Это одна точка
на веере 1 (topn=100) и эндпоинт веера 2 (K=25) — общий якорь.

Для каждого варианта V построить помесячный ряд активной доходности `d_t = r_V,t − r_base,t`
на общем окне. Оценить: средн. активная доходность (annualized), tracking error,
information ratio, terminal-wealth ratio, разница Sharpe (V − base).

**Инференс — bootstrap, не t-тест.** scipy/statsmodels в стеке НЕТ (только numpy+pandas).
Месячные доходности автокоррелированы, негауссовы, гетероскедастичны — обычный t-тест
неприменим. Использовать **stationary / circular block bootstrap** (чистый numpy):
ресэмплить блоки месяцев (длина блока ~√n или по автокорреляции), бутстрэп-распределение
для mean(d), IR и разницы Sharpe; двусторонний bootstrap p-value + 95% percentile-CI.

**Поправка на множественные сравнения обязательна.** 11 (веер 1) + 11 (веер 2) = 22
теста против одного baseline → Benjamini-Hochberg FDR на p-values, репортить
adjusted-p. Варианты сильно скоррелированы (общие холдинги) — тесты не независимы.
Минимум — BH-FDR; правильнее — White Reality Check / Hansen SPA (max-статистика через
тот же stationary bootstrap) для честного теста «лучший вариант значимо бьёт baseline
с учётом перебора». Без этого «победитель» — data-snooping артефакт.

**Реалистичное ожидание (заявить сразу):** ~156 месяцев, IR реалистично ≈0.3/год →
t ≈ 0.3·√13 ≈ 1.1. Мощности нет. Скорее всего НИ ОДИН вариант не переживёт поправку.
Это и есть ответ: по доходности topn статистически неразличимы → выбирать по обороту /
ликвидности / capacity, а не по NAV-кривой. Деливерабл ценен тем, что показывает это,
а не тем, что найдёт «значимого победителя». Репортить эффекты + CI, а не только p.

  концентрации — K≈20 из top-100 даёт лучший IR в выборке, но это in-sample,
  не значимо out-of-sample.
