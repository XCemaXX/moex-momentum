# MVP: Q1 monthly diff on q_history

Узкий слайс из `001_quartile_transitions.md`: «что купить, что продать в этом месяце, если держишь Q1». Делается сразу, не дожидаясь полного transitions-фреймворка (Sankey, stickiness-метрики).

## Что показывать

На странице `q_history.html`, **отдельной секцией под 4-Q-картами**, блок «Изменения Q1 vs прошлый месяц».

- **Купить (вошли в Q1):** список тикеров, для каждого — где был в предыдущем месяце:
  - `был в Q2` / `был в Q3` / `был в Q4`
  - `новичок` — отсутствовал во всех квартилях прошлого месяца (свежий тикер в универсе).
- **Продать (вышли из Q1):** список тикеров, для каждого — где он в текущем месяце:
  - `ушёл в Q2` / `ушёл в Q3` / `ушёл в Q4`
  - `делистнут / выпал из универса` — отсутствует во всех квартилях текущего месяца.

Для самого первого месяца (нет prev) секция показывает заглушку «нет данных за предыдущий месяц».

## Реализация

- Только JS-логика в `q_history.html`. `data.json` уже содержит все 4 квартиля на каждый месяц — диффа на клиенте достаточно.
- При смене месяца в dropdown: пересчёт diff между `data.holdings[month]` и `data.holdings[months[i-1]]`.
- Никаких изменений в `site_builder.py` / `plotly_charts.py` / data shape.

## Тесты

- Тест, что `q_history.html` содержит маркеры diff-секции (id блока, ключевые тексты).
- Опционально: вытащить diff-логику в standalone JS-модуль и протестить через `pytest-playwright` или просто `jsdom` — пока **нет**, добавим если усложнится.

## Не входит в эту задачу

- Sankey-диаграмма перетекания Q1↔Q2↔Q3↔Q4 — это `001_quartile_transitions.md`.
- Stickiness-метрики (экспоненциальное затухание, топ-10 «упёртых») — там же.
- Diff для Q2/Q3/Q4 — пока only Q1, расширение тривиальное при необходимости.
