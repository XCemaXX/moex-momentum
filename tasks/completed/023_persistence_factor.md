# Persistence-фактор: стабильность ранга как сигнал — DONE (H1 отвергнута)

> Закрыта 2026-05. Сигнал исследован до ворот и **забракован как стратегия**
> (H1 reject). Исследовательский код (`P`, отбор, буфер, NAV-раннер, ворота)
> был построен, прогнан и **удалён** — мёртвая гипотеза не остаётся в дереве.
> Этот файл — замороженный вердикт: цифры ниже **и есть** запись, скрипта для
> их повторного прогона больше нет. Виз (`task 001`) использует tenure, а не `P`.

## Что такое был `P` (для интерпретации цифр)

Кросс-секционный momentum-score (тот же `universe_at` + signal движка, NaN вне
юниверса), затем:

- `u = percentile_rank − 0.5` ∈ (−0.5, +0.5] — центрированный ранг месяца, верх → плюс.
- `P_t = α·u_t + (1−α)·P_{t−1}`, `α = 1 − 0.5^(1/h)`, half-life `h` в месяцах. Сид
  `P_0 = 0` (новичок «зарабатывает» персистентность за ~h мес — warm-up).
  **Гэп сбрасывает EWMA** (зеркалит gap-restart юниверса/pending, `task 008`):
  NaN-месяц рвёт run, возврат стартует с нуля.
- Отбор: top-N по `P` + **hard-exit** по сырому рангу (асимметрия — мягкий вход
  по `P`, жёсткий выход по рангу < floor).
- Бейзлайн оборота: **float-buffer** (MSCI/S&P) — купить при входе в top-ADD,
  держать пока в top-KEEP; учебниковый low-lag способ резать оборот.

Параметры исследования: `h=13.5` (≈ λ=0.95 из `task 001`), exit-floor 0.5,
top-N 25, buffer ADD/KEEP = 25/35. NAV считался на таймингах движка
(`backtest.py`), валидирован на curve_fit top-25 → коммиченный Q1 NAV = **12.757**.

## Ворота — ВЕРДИКТ (вся история 2013-01..2026-04)

- **Gate 1 — overlap:** медианный Жаккар(top-25 `P`, curve_fit Q1) = **0.471**
  (<0.6). Сигнал **не коллинеарен** — корзина реально другая. Тривиальный нулл
  («это просто моментум» на уровне членства) отвергнут. → backtest-worthy.
- **Gate 2 — self-turnover:** `P` 0.225 vs curve_fit 0.408 — оборот **−45%**,
  и `P` бьёт буфер (0.300). **H2 подтверждена.**
- **НО net NAV (13 лет):** `P` **6.93** vs curve_fit **12.76** — половина
  терминального богатства. Экономия оборота ≈ 0.18/мес × 5 б.п. ≈ **0.1%/год** —
  на порядки меньше, чем стоит лаг сглаживания.
- **Свип h∈{3,6,13.5,24,48}:** NAV держится в **6.6–8.0**, ни на одном h не
  подходит к 12.76, оборот почти не двигается (0.20–0.23). Отставание
  **структурное** (нулл Кархарта 1997: persistence = лагированный моментум),
  не вопрос настройки.
- **Буфер доминирует:** 12.62 @ 0.30 против `P` 6.93 @ 0.225 на фронте
  (alpha, turnover). По собственному критерию задачи — «если persistence не бьёт
  buffer по компромиссу — он не нужен» — **persistence не нужен.**

**Итог: H1 отвергнута. Persistence — не стратегия.** Если цель «торговать меньше
без потери доходности» — её в лоб решает однопараметрический буфер.

## Что НЕ делалось (осознанно)

- **CLI-команда / персист `P` на диск** — мёртвая теория не заслуживает
  production-команды и артефакта на диске; виз красит по tenure, не по `P`.
- **Internal-веер / sweep-страница** — полировка заведомо проигравшего сигнала.
- **Сохранение кода** — `persistence.py`, ворота, тесты удалены после вынесения
  вердикта (решение автора: правок в production от отклонённой гипотезы не остаётся).

## Связь с task 001

`task 001` (Sankey + «упёртые») задумывался потребителем `P`. По факту stickiness
= **самый длинный непрерывный tenure в квантиле** (буквальное определение
«упёртого» из task 001 §2) — прямее и нагляднее `P`, и не требует мёртвого
сигнала. Поэтому `P` в виз не пошёл.
