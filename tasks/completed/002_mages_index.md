## Зафиксированные решения

- **Бенчмарк = MCFTRR** (MOEX total-return, `data/indices/MCFTRR.csv`). Совпадает с нашими total-return кривыми (дивиденды с налогом 13%). IMOEX (ценовой) не ингестился и не нужен.
- **Месячная сетка.** Дневного движка нет; держим всё на `Period[M]` (закрытие последнего торгового дня), как базовый Q1. Дневная кривая магов на отдельной сетке сломала бы сравнение с месячным Q1 — не делаем.
- **Ротация = C1 (по календарю).** Снимок квартала, опубликованный в янв/апр/июл/окт, действует с того же месяца и далее (3 месяца). ~10 дней заглядывания вперёд на стыке кварталов — принято как погрешность второго порядка (соседние списки магов сильно перекрываются). Задокументировать в `methodology.md`.
- **Equity-кривая магов = только акции** (`pct_shares_only`). Облигации/валюта/фонд (`other`) в кривую не входят. Внутри квартала держим, ребаланс на стыке кварталов с дрейфом весов внутри квартала.
- **Внебиржевые акции** (NOMP/NOMPP/VMTP/BESKP — нет ценовых рядов) определяются по `canonical=null` (резолв дал тикер, но его нет в ценовом словаре) и на ингесте уходят в `other[]` с `type:"otc"` (с сохранением `ticker`). Поэтому `shares[]` = только торгуемые акции, `pct_shares_only` нативно даёт 100% по `shares[]`, а кривая не делает drop+renorm.
- **`N` и `mean_m` = `len(shares[])`** (все торгуемые акции). `other[]` (облигации/валюта/фонд/otc) не участвуют.
- **Метод взвешенного Q1 — аддитивный λ-тилт** (единственный). Дискретный winner-take-all отброшен; при желании концентрации добавим позже параметром (клип конвикции).
- **Код магов — отдельно.** Новый пакет `src/mages/` (sibling к `momentum/`, `ingest/` и т.п. — плоский layout без зонтика). Переиспользуем `load_panel`/панель доходностей из `momentum`, не дублируем.

## Методика

### Взвешенный Q1 — аддитивный λ-тилт (основной кандидат)

Та же выборка имён, что и базовый Q1 (равновес `base_i = 1/K`), веса смещены премией за упоминание у магов. Не упомянутое — **пол**; любое упоминание строго выше пола (упоминание = эндорсмент):
```
conviction_i = mi / mean_m   для упомянутых и торгуемых; иначе 0   # mi = pct_shares_only
mean_m = (Σ mi) / N          # среднее по торгуемым акциям магов
factor_i = 1 + λ · conviction_i                                    # пол = 1 при conviction=0
w_i ∝ base_i · factor_i ,  нормировка Σ w = 1
```
- `N`, `mean_m` — **только торгуемые акции** (`shares[]` минус внебиржа). `conviction = mi/mean_m` инвариантно к перенормировке.
- Не упомянут → `factor=1` (пол); упомянут → `factor>1`. Малое упоминание чуть выше пола, **никогда ниже** — чинит баг степенной формы `conviction^λ`, где `mi < 100/N` уводило ниже неупомянутых.
- `λ=0` → чистый Q1; `λ` растёт → премия за упоминание сильнее. Монотонно по `mi`, без разрыва. Дефолт λ=1 (под вопросом).
- Финансируется пропорционально со всех (нормировкой) — непрерывная версия идеи «раздать на остаток» без обрыва на `S'>100`.

Интерфейс: `weight(base_q1, mages_quarter, lam) -> {ticker: weight}`.

### Equity-кривая (магов и взвешенного Q1)

- Месячная доходность портфеля = `Σ w_i · total_return_i(t)` по панели `data/momentum/monthly/`.
- **Магов:** веса = `pct_shares_only` тек. квартала (перенормир. на торгуемые акции), ребаланс на стыке кварталов, дрейф внутри.
- **Взвешенный Q1:** каждый месяц берём Q1-состав этого месяца (`data/momentum/{signal}/holdings/`) + конвикции тек. квартала магов → веса по выбранному методу. Q1 меняется помесячно, конвикции — поквартально. Так кривая остаётся на той же месячной сетке, что и базовый Q1 → корректное сравнение.
- **MCFTRR:** месячные закрытия из `data/indices/MCFTRR.csv` (или готовый столбец из `q_values.csv`).
- Все кривые rebased к 1.0 на старте окна (как `_rebase_buttons`).

## Обновление (новый квартал → сайт)

Живой сайт собирается в CI (`pages.yml`) на push в `main`, а не из закоммиченного
`docs/pages/` (он не деплоится). CI пересчитывает monthly + backtest из сырья и
делает `site build` в `_site`. Страница магов читает `data/mages` вживую.

1. Скилл `parse_mages_index` пишет новый `data/mages/<YYYY-Qn>.json` (+ алиасы в
   `data/tickers.json`, если нужно).
2. **Предусловие:** месяцы квартала должны быть покрыты ценами/дивами/индексом и
   бэктестом. Если период новый — сперва обычный месячный апдейт данных (ingest),
   закоммитить сырьё. Если квартал в уже загруженных месяцах — шаг не нужен.
3. Закоммитить `data/mages/*` (+ `data/tickers.json`) и `git push` в `main` →
   CI пересоберёт и задеплоит. `docs/pages` коммитить не нужно.
4. Локальный preview: `momentum site build` (если добавлены и новые месяцы —
   сперва `momentum compute monthly` + `... backtest --signal curve_fit`).

Кривая/таблица удлиняются ровно настолько, насколько хватает holdings и доходностей
этих месяцев.

## Решено

- **λ = 1** (линейный тилт, по принципу). Без оптимизации по доходности.
- **Порог микропозиций** — не вводим, берём все упоминания.
- **Линия total-mages с облигациями** — не делаем (нет данных по облигациям; `other[]` только для исключения из акций).
